## Stage 1: Brainstorming

Explore design options and document them.

### Output File:
docs/superpowers/specs/DESIGN-{workflow_run_id}.md

### Tasks:
1. Read `brainstorming` skill

2. Research context:
   - Read `devflow` skill
   - Explore existing codebase
   - Check similar implementations

3. Create design document

4. Document 2-3 approaches:
   ## Approach A: [Name]
   - Description
   - Pros
   - Cons
   - Effort estimate

   ## Approach B: [Name]
   ...

5. Present to user:
   - Summarize each approach
   - Explain recommendation with rationale
   - Ask ONE question at a time

6. Self-review:
   - No TBD/TODO left
   - All options have clear trade-offs

### Gate:
- User approves chosen approach
- DESIGN-{workflow_run_id}.md written
