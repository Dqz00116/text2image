## Technical Approach

### Pattern

All Click option defaults change from hardcoded literals to `None`, with resolution at function body:

```python
@click.option("--model", default=None, help="Model ID (env: TEXT2IMAGE_MODEL)")
def generate(model, ...):
    load_dotenv()
    model = model or os.environ.get("TEXT2IMAGE_MODEL", "openai/gpt-5.4-image-2")
```

### Files Changed

**cli.py** — `generate` command:
- `--model`: `default=None` → resolve via `TEXT2IMAGE_MODEL` / `openai/gpt-5.4-image-2`
- `--output`: `default=None` → resolve via `TEXT2IMAGE_OUTPUT` / `output.png`
- `--aspect-ratio`: `default=None` → resolve via `TEXT2IMAGE_ASPECT_RATIO` / `1:1`
- `--image-size`: `default=None` → resolve via `TEXT2IMAGE_IMAGE_SIZE` / `1K`
- `--api-base`: `default=None` → resolve via `OPENROUTER_API_BASE` / `https://openrouter.ai/api/v1`

**cli.py** — `describe` command:
- `--vision-model`: `default=None` → resolve via `DESCRIBE_VISION_MODEL` / `openai/gpt-4o`
- `--backend`: `default=None` → resolve via `DESCRIBE_BACKEND` / `kimi`
- Default prompt: resolve via `DESCRIBE_DEFAULT_PROMPT` / built-in text
- Pass `api_base` to `create_describer("openrouter", ..., base_url=api_base)`

**describer.py** — `OpenRouterVisionDescriber.__init__`:
- Already has `base_url` param with default; `create_describer` already passes `base_url`. No change needed to the class itself.

**describer.py** — `create_describer`:
- For `openrouter`: pass `base_url` from kwargs (already supported)
