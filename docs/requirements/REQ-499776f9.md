## Description

Current CLI only supports single `--image` input. Upgrade to accept multiple images per request, enabling both character reference + style references to be passed together.

## Acceptance Criteria

- [ ] `--image` accepts multiple values: `-i a.png -i b.png -i c.png`
- [ ] Client `generate()` accepts `image_paths: list[str] | None`
- [ ] Multiple images are added as separate `image_url` parts in the message content array
- [ ] Backward compatible: single `--image` still works
- [ ] All existing tests pass
- [ ] New test covers multi-image message building
