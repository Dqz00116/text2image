## Description

Current CLI hardcodes several defaults that should be configurable via `.env` / environment variables. Also `OpenRouterVisionDescriber` hardcodes its base URL independent of the shared `--api-base` / `OPENROUTER_API_BASE` setting. This task adds `.env` support for every configurable default across both `generate` and `describe` subcommands, and ensures `OpenRouterVisionDescriber` shares the same API base.

## Acceptance Criteria

### generate defaults → .env

- [ ] `TEXT2IMAGE_MODEL` env var → `--model` default (was hardcoded `openai/gpt-5.4-image-2`)
- [ ] `TEXT2IMAGE_OUTPUT` env var → `--output` default (was hardcoded `output.png`)
- [ ] `TEXT2IMAGE_ASPECT_RATIO` env var → `--aspect-ratio` default (was hardcoded `1:1`)
- [ ] `TEXT2IMAGE_IMAGE_SIZE` env var → `--image-size` default (was hardcoded `1K`)
- [ ] `OPENROUTER_API_BASE` env var → `--api-base` default (was hardcoded `https://openrouter.ai/api/v1`)

### describe defaults → .env

- [ ] `DESCRIBE_VISION_MODEL` env var → `--vision-model` default (was hardcoded `openai/gpt-4o`)
- [ ] `DESCRIBE_BACKEND` env var → `--backend` default (was hardcoded `kimi`)
- [ ] `DESCRIBE_DEFAULT_PROMPT` env var → default prompt text
- [ ] `DESCRIBE_TIMEOUT` already read from env, ensure `.env` loading applies

### Shared

- [ ] `OPENROUTER_API_BASE` respected by `OpenRouterVisionDescriber` (currently hardcoded)
- [ ] All existing tests pass (46/46)
