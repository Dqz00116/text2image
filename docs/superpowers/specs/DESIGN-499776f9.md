# Design: Multi-Image Support

## Approach (only viable option)

Extend existing `--image` from single value to `multiple=True`. Client's `_build_messages` already refactored to accept `list[str]`. Remaining: update `generate()` signature and CLI wiring.

## No alternative needed

This is a mechanical extension. The API already supports multiple `image_url` parts in one message content array.
